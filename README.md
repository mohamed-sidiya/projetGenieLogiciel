MINI-RAPPORT D’AVANCEMENT DU PROJET
Application de Gestion d’Absences Universitaires

1. Présentation générale du projet
   L’objectif du projet est de concevoir une application web permettant de digitaliser la gestion des absences dans un établissement universitaire. Cette solution répond à une problématique réelle : la difficulté de suivre les présences et absences dans des universités où le nombre d’étudiants et de professeurs augmente constamment.
   Le système vise à remplacer les méthodes traditionnelles basées sur le papier par un outil moderne, automatisé, fiable et sécurisé.

2. État d’avancement actuel
   À ce stade du projet, plusieurs livrables importants ont été réalisés :

– Vision du produit
Une vision claire a été rédigée, définissant :
• le problème à résoudre
• le public cible
• les objectifs fonctionnels
• la valeur ajoutée du produit
Cette vision constitue la base conceptuelle du projet.

– Définition des acteurs
Les trois acteurs principaux (Étudiant, Professeur, Superviseur) ont été décrits en détail :
• leurs accès
• leurs rôles
• leurs fonctionnalités
• leurs interactions avec le système

– User Stories
Les besoins fonctionnels ont été transformés en User Stories structurées :
• 4 User Stories pour l’étudiant
• 3 User Stories pour le professeur
• 3 User Stories pour le superviseur
Chaque User Story suit le format :
« En tant que…, je veux…, afin de… ».

– Backlog du produit
Un backlog clair et complet a été créé, comprenant :
• les User Stories
• les tâches (tasks) associées
• la priorité
• le statut Agile
Ce backlog constitue la base de la planification des sprints.

– Mise en place du Kanban
Les colonnes principales du Kanban ont été définies :
Backlog, To Do, In Progress, Done.
Les tâches peuvent désormais être suivies visuellement selon leur progression.

– Création des fichiers Markdown
Plusieurs documents techniques du projet ont été générés et structurés proprement pour une utilisation dans GitHub :
• vision_projet
• acteurs
• user_stories
• sprint_planning_1
• backlog (optionnel)

3. Méthodologie utilisée
   Le projet suit une approche inspirée de Scrum :
   • définition de la vision produit
   • création du Product Backlog
   • organisation du Sprint Planning 1
   • définition du Kanban
   • découpage en User Stories et tâches
   Ce modèle permet un travail organisé et une meilleure collaboration entre les membres de l’équipe.

4. Ce qui est prêt pour la suite
   L’équipe dispose maintenant de :
   • une base documentaire solide
   • un backlog exploitable
   • un cadre organisationnel (User Stories, Kanban, Sprint Planning)
   Tout est prêt pour démarrer officiellement le développement.

5. Prochaines étapes recommandées
   • commencer le Sprint 1 (développement du module Étudiant)
   • réaliser les maquettes des interfaces (UI/UX)
   • choisir la stack technique (ex. Laravel, Spring Boot, Node.js + React)
   • créer une structure propre du dépôt GitHub
   • préparer l’environnement de travail (IDE, outils de gestion, etc.)

6. Conclusion
   Le projet est actuellement bien structuré. La vision, les besoins, les acteurs, les User Stories et le Backlog ont été clairement définis. L’équipe peut désormais passer à la phase de conception technique puis au développement.
