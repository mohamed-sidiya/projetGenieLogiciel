1. L’ÉTUDIANT
Accès :
L’étudiant a accès à une interface personnalisée, sécurisée par un
identifiant et un mot de passe, lui permettant d’interagir avec le
système de présence.
• Fonctionnalités :
- Enregistrement de présence :
Il peut entrer le code unique fourni par le
professeur pour marquer sa présence en temps réel.
- Une notification instantanée : confirme la validation de sa
présence.
- Gestion des absences :
Consultation en temps réel de son taux d’absence,
```
affiché de manière claire et détaillée (par cours, par
```
```
semaine, ou par mois).
```
- Visualisation des historiques de présence : avec les dates et les
cours concernés.
- Justification des absences :
- Possibilité de déposer une excuse en ligne en cas d’absence
avec pièce jointe si nécessaire, comme un certificat médical
- Suivi de l’état de la justification (en attente, validée ou
```
refusée).
```
- Imprimer les absences :
- Possibilité de télécharger un document PDF ou EXCEL
contenant ses abences
- Notifications :
- Alertes en cas d’absence non justifiée ou de taux
d’absentéisme élevé.

2. LE PROFESSEUR :
Accès :
Le professeur dispose d’un espace dédié, accessible via des
identifiants sécurisés, lui offrant un contrôle complet sur la gestion
des présences de ses cours.
Fonctionnalités :
- Génération et partage du code :
- Chaque professeur possède un code personnel et
dynamique, régénéré automatiquement pour des raisons
de sécurité.
- Partage simplifié du code (affichage en classe, envoi
```
numérique, etc.).
```
- Enregistrement des présences :
- Validation automatique de sa propre présence dès qu’un
élève saisi son code.
- Peut aussi valider la présence manuellement.
- Visualisation en direct de la liste des élèves présents/
absents pendant le cours.
3. LE SUPERVISEUR :
Accès :
Le proviseur bénéficie d’un tableau de bord administratif complet,
centralisant toutes les données relatives aux absences des élèves
et des professeurs.
Fonctionnalités :
- Monitoring global :
- Consultation des statistiques d’absence pour

```

```
